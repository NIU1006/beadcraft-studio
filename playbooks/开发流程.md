# Playbook：豆趣工坊开发流程

## 目的
从零构建完整的拼豆图纸生成网页应用，用户可以上传图片、选择参数、生成可制作的拼豆图纸并下载。

## 前提条件
- Node.js >= 18
- npm 或 yarn 可用
- 色卡PDF文件已存在（pindou/拼豆色卡测295色品牌对照表.pdf）

## 步骤

### 阶段1：项目初始化
1. 【脚本调用】`scripts/init-project.sh` — 使用Vite初始化React+TypeScript项目
2. 【验证】检查项目可正常启动

### 阶段2：安装依赖
1. 【脚本调用】`scripts/install-deps.sh` — 安装UI、图表、PDF等依赖
2. 【验证】package.json 包含所有必要依赖

### 阶段3：色卡数据准备
1. 【脚本调用】`scripts/extract-colors.py` — 从PDF提取色卡数据为JSON
2. 【判断】若PDF提取失败 → 手动创建基础色卡JSON
3. 【验证】src/data/colors.json 存在且格式正确

### 阶段4：基础组件开发
1. 【编码】创建基础布局组件（Header/Main/Footer）
2. 【编码】创建图片上传组件（UploadArea）
3. 【编码】创建参数设置面板（SettingsPanel）
4. 【验证】页面布局正常显示

### 阶段5：核心功能实现
1. 【编码】实现颜色匹配算法（CIE76）
2. 【编码】实现图片像素化处理（Canvas）
3. 【编码】创建图纸画布组件（PatternCanvas）
4. 【验证】上传图片可生成像素图

### 阶段6：统计与导出
1. 【编码】实现材料统计组件（StatisticsPanel）
2. 【编码】实现PNG导出功能
3. 【编码】实现PDF导出功能
4. 【验证】导出文件可正常打开

### 阶段7：构建部署
1. 【脚本调用】`scripts/build.sh` — 构建生产版本
2. 【验证】dist目录生成，无错误

## 判断标准
- 色卡数据提取失败时 → 使用硬编码的基础颜色数据（Mard品牌约50色）作为fallback
- 图片处理性能不足时 → 限制最大处理尺寸为100x100，大尺寸提示用户缩小
- 浏览器兼容性问题时 → 降级处理，确保基础功能可用

## 验证
- [ ] 可上传图片并预览
- [ ] 可调节画板尺寸和豆子大小
- [ ] 可选择不同品牌色卡
- [ ] 生成图纸显示网格和色号
- [ ] 材料统计准确
- [ ] PNG和PDF下载正常
